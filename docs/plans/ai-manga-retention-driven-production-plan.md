# AI 漫剧 / 小说视频「留人导向」生产方案

## 一、调研结论：爆款短剧/漫剧在用什么手段留人

搜索了中文短剧、AI 漫剧、小说推文、海外 micro-drama 的资料后，核心手段可以归纳为一套**“钩子 + 情绪 + 悬念 + 节奏”**的四位一体模型：

| 手段 | 行业做法 | 数据/原则 |
|---|---|---|
| **黄金 3 秒钩子** | 前 3 秒必须出现冲突、羞辱、反转、悬念或强反差 | 72% 用户在 3 秒内划走；前 3 秒有肢体冲突的短剧完播率高 47% |
| **强情绪极端化** | 不是“有点难过”，而是“笑着笑着突然哭到抽搐”；不是“生气”，而是“笑着喝茶，桌上有把刀” | 情绪越极端，停留越长 |
| **信息给一半藏一半** | 钩子要露线头，让观众主动脑补 | 主动参与感比直接给答案更上瘾 |
| **每集结尾卡钩子** | 身份曝光、反转台词、惊天秘密、危险降临、绝地反击预告 | 粉丝回访率可提升 67% |
| **压抑 → 爆发 → 奖赏** | 主角先被欺压（代入），再反转（爽），最后反派受惩（宣泄） | 男频/女频通用的情绪链条 |
| **旁白占 80% 以上** | 小说推文/漫剧解说普遍不用对白，或用旁白带过 | 避免对口型、多音色、节奏断裂 |
| **节奏呼吸感** | 每 5-8 个镜头插 1 个“呼吸镜头”（空镜、微动作、物件特写） | 打破密集感，让观众不疲劳 |
| **能量曲线上升** | 单集信息密度递增，结尾最高；整部剧情绪螺旋上升 | 每 15 秒一个信息点爆发 |

关键认知：短剧/漫剧不是电视剧，也不是电影。**它是“情绪产品”**，核心目标不是把故事讲完，而是让观众“停不下来”。

## 二、我们的生产目标

从“把故事讲清楚”升级为**“让观众 3 秒停下、30 秒入戏、结尾必须点下一集”**。

具体指标：
- 每集开头 3 秒内必须有钩子。
- 每集结尾 5 秒内必须有悬念钩子。
- 每集至少 1 次情绪反转，理想 2-3 次。
- 单集时长 60-120 秒（首集可 2-3 分钟做世界观钩子）。
- 无对白 / 纯旁白模式作为默认。

## 三、全链路改造方案

### 1. 前期：选题与分集策略

#### 1.1 全局钩子设计（premise hook）
在创建 drama 时，让 LLM 先输出一个“一句话核心钩子”：
> 例：丈夫把弟媳捧成女主人，真正的妻子笑着递上一份离婚礼物。

这个钩子要写入 `dramas.hook`，用于：
- 首集开篇旁白。
- 剧集封面/标题。
- 所有宣发文案。

#### 1.2 分集节奏模板
不再只是“把原文切成等长段落”，而是按**情绪弧线**分集：

```
开篇期 (Ep 1-3)：快速建立人物关系 + 核心矛盾 + 第一个大反转
升温期 (Ep 4-?)：矛盾升级，每集一个小高潮
高潮/反转期：身份揭露、情绪爆发
收尾期：真相大白或开放式结局
```

每集必须包含：
1. **Opening Hook**（开头钩子）
2. **Immediate Conflict**（立即冲突）
3. **Mid-episode Turn**（中段转折）
4. **Emotional Push / Reveal**（情绪推进或揭示）
5. **End Cliffhanger**（结尾悬念）

#### 1.3 单集三段式节奏
```
开头 0-15s：钩子 / 上集悬念承接 / 冲突预览
中段 15s-90s：冲突推进 / 反转 / 打脸 / 情感升级
结尾 最后 15s：新悬念 / 情绪爆发点 / “未完待续”感
```

### 2. 分镜阶段：把情绪变成镜头

#### 2.1 镜头能量分级
给每个镜头标记能量等级，控制节奏起伏：
- **高能镜头**：冲突、反转、爆发、核心动作（短镜头 3-6s，快节奏）
- **中能镜头**：对话、犹豫、观察、情绪积累（中镜头 5-10s）
- **低能/呼吸镜头**：环境、微动作、静默反应、物件特写（5-8s，让观众喘口气）

规则：每 5-8 个镜头必须插 1 个呼吸镜头；高能镜头不要连续超过 3 个。

#### 2.2 开头钩子镜头（前 3 秒）
第一镜必须是强视觉冲击，常见做法：
- 快速推近 → 急停 → 固定（面部特写/关键物件）
- 反常识画面：笑着举杯的女人、被当作女主人的弟媳、真正的妻子站在暗处
- 直接抛出悬念：金色礼盒、离婚协议书、亲密合照

#### 2.3 情绪外化镜头
禁止“他很震惊”“她很痛苦”这类笼统描述，改为可拍的微动作：
- 手指收紧、呼吸变重、眼神停顿、喉结滚动、嘴角压紧、后退半步、手停在半空
- 物件特写：停在半空的杯子、捏皱的纸、亮起的手机屏幕

#### 2.4 结尾悬念镜头
最后一镜必须是：
- 信息揭晓但后果未明（盒中离婚协议）
- 新人物/新危机出现
- 主角一个意味深长的表情/动作
- 旁白直接问出一个让观众想知道答案的问题

### 3. 旁白阶段：电影解说式讲述

旁白要同时完成三个任务：**交代信息、带情绪、埋钩子**。

#### 3.1 每集旁白结构
```
第一句（钩子）：直接抛出最大冲突/反差
前 15 秒：人物关系 + 当前处境
中段：事件推进 + 情绪升级
结尾：揭晓关键信息 + 甩出下一集问题
```

#### 3.2 旁白示例（对比旧版）

**旧版（碎片化）：**
> 他再次默许别人叫她宋太太。这一次，我没闹，只是乖顺举杯。

**新版（电影解说式）：**
> 宋峥延是林砚的丈夫，却默许全场宾客把弟媳苏轻轻称作宋太太。林砚没有闹，只是笑着举杯附和——反常的平静，往往比失控更危险。

**旧版（没信息）：**
> 他猛地抬头，眼里的错愕压都压不住。

**新版（带关系和原因）：**
> 听到林砚亲口喊出那声“宋太太”，宋峥延猛地抬头。他习惯了妻子的歇斯底里，却没料到她能这样笑着把位置让出去。

#### 3.3 结尾钩子旁白
每集最后一句旁白必须是悬念：
> 他不知道，那个金灿灿的礼盒里，躺着的不是上市礼物，而是一纸早就签好字的离婚协议。
> 她到底是谁？为什么所有人都怕她？
> 真正的宋太太，终于开始反击了。

### 4. 声音设计：用 BGM 和音效托情绪

#### 4.1 音乐跟随能量曲线
- 开头钩子：音乐骤起或突然静止后进入紧张氛围
- 压抑铺垫：低沉弦乐/环境音
- 情绪转折：鼓点加入、弦乐上扬
- 反转爆发：音乐骤强或节奏加快
- 结尾钩子：音乐在最高潮处突然收住，或留一个悬疑长音

#### 4.2 音效锚点
- 关键动作：按键声、纸张声、关门声、杯子放下声
- 情绪转折：心跳声、金属摩擦声、环境音突然消失
- 悬念钩子：电话铃声、短信提示音、突然的安静

### 5. 剪辑与合成：节奏是最后的情绪放大器

#### 5.1 镜头时长由情绪和旁白共同决定
- 高能镜头：短，3-6 秒
- 情绪特写：可略长，让观众看清微表情
- 呼吸镜头：5-8 秒，不要停太久
- 结尾钩子：最后一镜可适当延长 1-2 秒，让悬念停留

#### 5.2 转场规则
- 动作顺接：上一镜人物抬手，下一镜手已经落下
- 视线顺接：上一镜人物看向某处，下一镜展示他看到的
- 情绪对比：热闹场景切孤独特写
- 音乐/音效卡点：关键动作落在鼓点上

#### 5.3 字幕节奏
- 字幕随旁白逐句出现，不要一次性铺满。
- 关键金句可单独停留 1-2 秒。
- 结尾钩子字幕停留到下一集提示出现。

### 6. 封面与标题（前端展示）

每集生成时同步输出：
- **标题**：2-8 字，带钩子（例：宋太太、金盒之下、不再打扰）
- **封面文案**：一句话钩子（例：真正的宋太太，笑着递出了离婚协议）
- **缩略图提示词**：选择本集最具冲突的一帧，叠加标题

## 四、系统层改造清单

### 1. 数据模型
- `dramas.hook`：一句话核心钩子。
- `episodes.opening_hook`：本集开头钩子文案。
- `episodes.cliffhanger`：本集结尾悬念文案。
- `episodes.energy_curve`：JSON，记录每镜能量等级。
- `episodes.dialogue_mode`：`narration_only`（默认） / `with_dialogue`。

### 2. Agent/Prompt 改造
- **episode-splitter**：按情绪弧线 + 钩子/悬念分集，每集输出 opening_hook 和 cliffhanger。
- **storyboard_breaker**：
  - 为每镜标注能量等级（high/medium/low）。
  - 第一镜必须设计为前 3 秒钩子。
  - 最后一镜必须设计为结尾悬念。
  - 加入呼吸镜头规则。
  - 无对白模式下不输出 dialogue。
- **storyboard_splitter**：不再按对白拆分，按动作阶段/情绪转折/空间转移拆分。
- **narrator**：
  - 第一句必须是 opening_hook。
  - 中间交代人物关系 + 事件 + 情绪。
  - 最后一句必须是 cliffhanger。
  - 无对白模式下把对话改写成旁白。
- **视频 prompt 生成**：加入能量等级对应的运镜、光影、动态元素。
- **compose**：按能量等级和旁白长度决定镜头时长，BGM/音效节奏匹配。

### 3. 前端 UI
- 分集页显示每集的 `hook` 和 `cliffhanger`。
- 分镜页显示每镜的能量等级和“是否为钩子/悬念/呼吸镜头”。
- 提供“重新生成钩子”按钮，单独优化首集/全剧钩子。
- 播放页结尾显示“下一集预告”卡片。

### 4. 默认配置
- 新 drama 默认开启 `narration_only` 模式。
- 新 episode 默认生成 opening_hook 和 cliffhanger。
- 首集时长放宽到 2-3 分钟，用于世界观钩子。

## 五、实施步骤

### 第一阶段：无对白 + 钩子/悬念骨架（MVP）
1. 新增 `dialogue_mode` 字段，默认 `narration_only`。
2. 重写 narrator prompt：电影解说式、第一句钩子、最后一句悬念、交代人物关系。
3. 改造 storyboard_breaker：为每集强制生成 opening_hook 和 cliffhanger，第一镜和最后一镜按钩子/悬念设计。
4. 改造 compose：旁白-only 时长计算，结尾悬念多留 1-2 秒。

### 第二阶段：情绪节奏与声音设计
5. 给 storyboard 增加 `energy_level` 字段。
6. 在 storyboard_breaker/splitter 中实现呼吸镜头规则。
7. 为 BGM/sound_effect 生成增加能量曲线匹配逻辑。
8. 在视频 prompt 中加入运镜公式（钩子开头、反转揭示、呼吸镜头等）。

### 第三阶段：数据与运营层
9. 每集生成封面标题、封面文案、缩略图提示词。
10. 前端展示 hook/cliffhanger/能量曲线，支持单独重跑钩子。
11. A/B 测试：同一集生成两个版本钩子，用数据反馈优化。

## 六、风险与应对

| 风险 | 应对 |
|---|---|
| 钩子太大后续圆不上 | LLM 生成钩子时同步检查后续是否有回收，低于 30% 出现频率判为无效钩子 |
| 过度追求反转导致逻辑断裂 | 保留“剧情完整性”硬约束，反转必须基于已有伏笔 |
| 高能镜头太密观众疲劳 | 强制每 5-8 镜插入呼吸镜头 |
| 无对白丢失角色金句 | 关键金句由解说音色读出，作为旁白的一部分 |
| 节奏过快情感铺垫不足 | 低能镜头和 BGM 留白承担情绪铺垫功能 |

## 七、预期成片效果

以 drama 18 为例，改造后的一集可能呈现为：

> **0-3s 钩子**：镜头急推到林砚举杯微笑的脸，旁白第一句：“宋峥延是林砚的丈夫，却默许宾客把弟媳称作宋太太。”
> **3-30s 冲突建立**：宋峥延错愕、试探、林砚平静回敬。
> **30-80s 升级**：苏轻轻以女主人姿态登场，全场欢呼，宋峥延冷眼等待林砚失控。
> **80-100s 反转**：林砚递上金色礼盒，宋峥延以为她认命。
> **最后 5s 悬念**：镜头穿透礼盒，离婚协议书静静躺在里面。旁白：“签名栏的墨迹，早就干了。”

观众看完后唯一想做的事：点下一集。
