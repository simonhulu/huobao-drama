# 无对白 / 纯旁白解说模式方案

## 调研结论

搜索了 AI 漫剧、小说推文、短剧自动化的公开资料后，主流做法分为两类：

1. **解说类二创（占大多数）**
   - 全程以旁白讲解为主，旁白占比 80% 以上。
   - 人物对话不单独拆分，只由旁白用“他说了什么 / 她回了什么”一带而过，或干脆不出现。
   - 画面只服务情绪和情节推进，靠 BGM + 音效 + 镜头语言讲故事。
   - 参考：抖音/快手小说推文号、B 站漫剧解说。

2. **纯视觉高级模式**
   - 无字幕、无对白、无旁白，完全靠画面、动作、表情、环境音效推进。
   - 对视频生成质量要求极高，目前 AI 很难稳定做到表情和动作连续，工业化批量难度大。

结合你的需求（“有对白好像很奇怪，可以不需要对白，只完全围绕着情绪和情节”），**最适合现在走的是“解说类二创 / 纯旁白模式”**：保留一个高质量解说音轨，去掉所有角色原声对白。

## 为什么要去掉对白

当前系统的问题：
- 对白需要从原文逐字提取，很容易归属错误、语气不对、口型对不上。
- 多角色配音需要多个音色，TTS 稳定性差，情绪常常不到位。
- 对白和解说在抢信息：解说刚要讲情节，角色突然开口，节奏被打断。
- 用户听的是“大女主”音色，突然切到男声/女声对白，情绪连贯性被破坏。

去掉对白后：
- 整集只用一个解说音色（大女主），情绪统一。
- 旁白可以自由地把对话内容改写成故事讲述，不用“让位”。
- 分镜不再需要对嘴型、分角色配音，生成和合成更简单。
- 观众听到的就是一段完整的故事解说，像电影解说号一样自然。

## 目标

新增一个 **“无对白模式”**（No-Dialogue / Narration-Only Mode）：
- 分镜阶段不再输出角色 `dialogue`。
- 旁白阶段把原本的对白内容转化成解说词的一部分。
- TTS 只生成解说音频，不生成角色对白音频。
- 合成时镜头时长由解说音频决定，不需要等待对白。

## 具体改造点

### 1. 数据层：新增 `episodes.dialogue_mode` 字段

- 枚举：`narration_only`（纯旁白，默认） / `with_dialogue`（保留原声对白）。
- 也可以加到 `dramas` 层作为剧集默认。
- 数据库迁移 + schema.ts + 原始 SQL 初始化语句同步更新。

### 2. 分集/分镜阶段

#### `storyboard_breaker`
- `narration_only` 模式下：
  - 仍然读取原文里的对话，但把对话内容作为“情节信息”理解，不写入 `dialogue` 字段。
  - 镜头 focus 从“对白回合”转移到“反应、动作、情绪、关键物件”。
  - 增加要求：每个镜头都要想清楚“如果观众听不到角色说话，这个画面能不能把情绪和情节传过去”。

#### `storyboard_splitter`
- `narration_only` 模式下：
  - 不再按“对白回合”拆分镜头。
  - 只按“动作阶段、情绪转折、空间转移、信息功能切换”拆分。

### 3. 旁白阶段：`narrator`

新增“无对白解说稿”规则：

- 旁白就是完整的故事讲述，像电影解说一样。
- 原文里的关键对话，由旁白用第三人称转述：
  - 错误：`林砚说：“宋太太呀。”`
  - 正确：`林砚笑着回了他一句——宋太太呀。`
- 直接引用原文金句时，只保留最有力的 1-2 句，且由解说音色读出，而不是角色音色。
- 不再遵守“有对白就让位”的规则；相反，要把对白信息消化进解说里。
- 每镜头 1-3 句，整体像一段连续的故事音频。

### 4. TTS 阶段

- `narration_only` 模式下：
  - `scheduleTTSForEpisode` 和 `POST /episodes/:id/generate-tts-all` 只检查 `narrationAudioUrl`。
  - 跳过所有 `dialogue` 相关音频生成。
  - 每个镜头只生成一条 `narration_audio_url`。

### 5. 合成阶段：`ffmpeg-compose`

- `narration_only` 模式下：
  - 镜头时长 = `narration_audio` 长度 + 少量留白（0.3-0.5s）。
  - 不需要等 dialogue 音频，简化等待逻辑。
  - BGM 和音效轨道保持，用音乐和音效补情绪。

### 6. 视频提示词

- `narration_only` 模式下，video_prompt 中 `<voice>` 标签只用于标记“解说画外音”，不标记角色对白。
- 强化动作、表情、微细节、环境动态的描述，让画面本身会说话。

### 7. 前端 UI

- 在分集页/剧集页增加开关：
  - `无对白模式（纯旁白解说）`
  - `保留原声对白`
- 默认建议 `无对白模式`。
- 切换模式时提示会清空分镜/旁白/音频/成片并重新生成。

### 8. 向后兼容

- 现有集默认保持 `with_dialogue`，老用户不强制切换。
- 新集默认 `narration_only`。
- 用户随时可以在单集页面切换。

## 预期效果示例

以 drama 18 / episode 1 为例，无对白模式下旁白可能变成：

> “宋峥延是林砚的丈夫，却默许全场宾客把弟媳苏轻轻称作宋太太。林砚没有闹，只是笑着举杯附和。宋峥延猛地抬头——他习惯了妻子的失控，却没料到她能这样平静地把位置让出去。他故意停住话头，等林砚发作。可林砚只是点了点头，把他和弟媳去纽约跨年的行程默认了下来。宋峥延不死心，又搬出让苏轻轻代言新品牌的借口。林砚没有反驳，只回了一句——这是峥嵘的遗愿。宋峥延自己最常用的挡箭牌，被她接过来，反手刺了回去。”

整段就是一个完整的故事音频，没有角色突然插嘴，情绪由解说声音和画面一起推进。

## 需要改动的主要文件

1. `backend/src/db/schema.ts`：新增 `episodes.dialogue_mode`。
2. `backend/src/db/index.ts`：同步原始建表 SQL。
3. `backend/src/agents/index.ts`：
   - `storyboard_breaker` 增加无对白规则。
   - `storyboard_splitter` 增加无对白拆分规则。
   - `narrator` 增加无对白解说稿规则。
4. `backend/src/services/tasks/auto-pipeline.ts`：`scheduleTTSForEpisode` 按模式过滤目标。
5. `backend/src/services/ffmpeg-compose.ts`：按模式简化时长计算和音频选择。
6. `backend/src/routes/episodes.ts`：
   - `PUT /episodes/:id` 支持 `dialogue_mode`。
   - `generate-tts-all` 按模式生成。
   - `generate-narration-audio` 可默认处理无对白模式。
7. `frontend/app/pages/drama/[id]/episode/[episodeNumber].vue`：增加无对白开关。
8. `skills/narrator/SKILL.md`：同步无对白解说规范。

## 风险

- 完全去掉对白会失去一些原文“金句”的角色语气；解决方案是保留极少数金句由解说音色读出。
- 如果解说写得太长，镜头画面会停留过久，需要继续控制每镜 1-3 句。
- BGM 和音效的情绪承载压力变大，需要后续加强音效设计。

## 建议的下一步

1. 先默认开启 `narration_only` 模式。
2. 用 drama 18 的某一集完整跑一遍：分镜 → 旁白 → TTS → 合成。
3. 你听一下成片，确认没有对白后情绪是否足够；如果不够，再强化 BGM/音效或回调部分金句。
