# 旁白解说指南

把真实故事讲得引人入胜，逐镜头撰写旁白，让观众**不用看原文也能听懂情节、被拉进情绪**。

## 核心方法：电影解说式讲述

旁白是面向观众的解说声，不是角色内心独白。优秀电影解说在讲每个镜头时都会同时做到：

1. **交代信息**：这是谁、和谁什么关系、现在什么情况
2. **讲清事件**：这个镜头里发生了什么、为什么重要
3. **带起情绪**：让观众感受到讽刺、紧张、委屈、爽感等

| 错误做法 | 正确做法 |
|---------|---------|
| 整篇故事切段配图 | 每个镜头看画面 + 回原文取料 |
| 照搬书面原文朗读 | 改写成口语化讲述 |
| 只剩碎片化情绪 | 讲清人物、关系、事件、情绪 |
| “我心里想”“我只觉得” | “他不知道，林砚等的就是这句” |

## 什么信息不能丢

- 人物身份与关系（尤其在开头或角色首次出场）
- 关键动作与决定
- 冲突升级与反转
- 情绪转折与落差
- 悬念钩子

背景交代、人物动机必须**依附于事件**，不能单独用旁白解释；更不能写成主角的内心独白。

## 三个信息来源（生成时都会给到）

1. **original_story**：原始故事原文 —— 情节、细节、金句的来源，保证真实有料
2. **formatted_script**：格式化剧本 —— 帮你确认场景顺序、对白关系、戏剧推进
3. **当前镜头语义**：`action / result / atmosphere / description / image_prompt / location / time / character_names` —— 告诉你这个镜头用什么来承载故事
4. **相邻镜头旁白**：保证承上启下、不重复、不剧透

## 撰写原则

1. **先让人听懂，再让人动情**
   - 不要假设观众知道前因后果。开头镜头必须交代核心人物关系和当前处境。
   - 例子：不要只说“他默许别人叫她宋太太”，要说“宋峥延是林砚的丈夫，却默许宾客把弟媳苏轻轻称作宋太太”。

2. **锚定镜头**
   - 旁白必须落在当前镜头承载的叙事任务上，不是泛泛复述。

3. **讲故事，不写内心**
   - 用第三人称或说书人视角描述事件、冲突、反转。
   - 严禁“我心里想”“我只觉得”“我告诉自己”这类纯内心独白。

4. **取真料**
   - 情节和细节来自原文，不凭空编造。

5. **可转译**
   - 不可直接看见的信息，要借当前镜头的表演、环境、物件、反应来讲出来，而不是解释心理。

6. **口语化**
   - 短句、有节奏、有代入感，像电影解说号而不是念书。

7. **连贯**
   - 参考前一镜头旁白承接，不重复已讲内容，不提前剧透下一镜头。

8. **给对白让位（硬规则）**
   - **有 dialogue（角色原声对白）**：解说只做情绪铺垫或解释对白分量，**严禁复述、改述或提前剧透**这句台词的内容——这条信息由角色原声承载，解说不得再讲一遍。
   - **无 dialogue**：解说承担该镜头的全部叙事信息，推进剧情。
   - **核心原则**：解说和原声对白是**分工关系，不是叠加关系**。同一条信息不要既被解说讲一遍又被角色说一遍。

9. **控时长**
   - 每条 1-3 句，与画面停留时长相称。信息量大或需要交代的镜头可以略长，但不要用长复合句堆砌。

## AI 漫剧旁白模式

当任务明确要求“AI 漫剧”“竖屏短剧”“快节奏”或输入内容来自 `ai_manga_drama` 风格的分集时，启用以下规则：

1. **紧凑但不失信息**
   - 每镜头旁白控制在 **1-3 句**。
   - 极速模式可以 1-2 句，但关键转折处必须让观众听懂前因。

2. **情绪优先**
   - 旁白用于放大情绪落差：讽刺、冷漠、爆发、释然。
   - 能用画面表现的信息，旁白一个字都不说。

3. **给对白和画面让位**
   - 有角色对白的镜头，旁白只铺垫或解释对白分量，严禁复述对白。
   - 无对白镜头，旁白也只补充最关键的情节推进点。

4. **口语化、短句化**
   - 避免长复合句，多用断句、反问、短打。
   - 像电影解说号现场叙述。

## 输出规范

- 每个镜头一条 `narration`，纯讲述文本
- 不带“旁白：”前缀
- 不写镜头语言、提示词、画面描述，只写要朗读的解说词

## 精稿直出 / 纪录片旁白模式

当任务或项目 workflow 为 `direct_script`（精稿直出）时，启用本模式：

1. **以 script_content 为权威来源**
   - 优先读取 `script_content`，其次 `original_story`
   - 按原文顺序逐镜头解说，不向原文添加未有的情节或悬念

2. **不强制钩子/悬念结构**
   - 第一句不一定是钩子
   - 最后一句不一定是 cliffhanger
   - 允许平铺直叙、背景交代、因果说明

3. **信息完整优先**
   - 人物关系、背景信息、上下文因果都可以讲
   - 不为了“情绪落差”而删减必要信息
   - 不强行制造反转或戏剧化冲突

4. **对白让位规则仍然适用**
   - 有角色原声对白的镜头，旁白只做铺垫或解释对白分量，严禁复述
   - 无对白镜头，旁白承担全部叙事

5. **语气**
   - 保持电影解说/纪录片解说口吻
   - 口语化、连贯、适合朗读

## 使用步骤

1. 调用 `read_narration_context` 读取原始故事和全部镜头
2. 通读原文，理解完整情节脉络
3. 逐镜头撰写旁白（看画面 + 取原文料 + 顾上下文）
4. 调用 `save_narrations` 一次性保存全部旁白
