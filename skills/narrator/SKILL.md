# 旁白解说指南

把真实故事讲得引人入胜,逐镜头撰写旁白,既体现故事精彩,又和画面天衣无缝。

## 核心方法:看图说书

旁白不是把原故事照念一遍,而是**针对每一张画面**,从原文里取出对应的情节,改写成讲给人听的解说词。

| 错误做法 | 正确做法 |
|---------|---------|
| 整篇故事切段配图 | 每个镜头看画面 + 回原文取料 |
| 照搬书面原文朗读 | 改写成口语化讲述 |
| 旁白讲到画面外的情节 | 只讲当前镜头看得到的内容 |

## 三个信息来源(生成时都会给到)

1. **original_story**:原始故事原文 —— 情节、细节、金句的来源,保证真实有料
2. **当前镜头画面语义**:`action / result / atmosphere / description / image_prompt / location / time` —— 保证旁白讲的就是观众看到的
3. **相邻镜头旁白**:保证承上启下、不重复、不剧透

## 撰写原则

1. **贴画面**:旁白内容必须落在当前镜头画面上,不跑到画面外的情节
2. **取真料**:情节和细节来自原文,不凭空编造
3. **口语化**:短句、有节奏、有代入感,像讲故事而不是念书
4. **连贯**:参考前一镜头旁白承接,不重复已讲内容,不提前剧透下一镜头
5. **给对白让位**:镜头若有角色原声对白(dialogue),旁白做铺垫或留白,不复述台词
6. **控时长**:每条 1-3 句,与画面停留时长相称,过长会让画面停太久

## 输出规范

- 每个镜头一条 `narration`,纯讲述文本
- 不带"旁白:"前缀,不带角色名
- 不写镜头语言、提示词、画面描述,只写要朗读的解说词

## 使用步骤

1. 调用 `read_narration_context` 读取原始故事和全部镜头
2. 通读原文,理解完整情节脉络
3. 逐镜头撰写旁白(看画面 + 取原文料 + 顾上下文)
4. 调用 `save_narrations` 一次性保存全部旁白
