# 超载镜头细分指南

## 什么是超载镜头

一个镜头的 `narration（旁白）+ dialogue（对白）` 总字符数超过阈值（默认 60 个字符），或单条 `narration / dialogue / action / description / result` 同时包含多个可见场景、动作阶段、对话回合、情绪转折、叙事功能切换时，即视为超载镜头。

超载镜头会导致单镜头信息密度过高、画面停滞、视频模型难以在 6-12 秒内完整表达内容，因此需要拆分为多个紧凑镜头。

## 拆分目标

- 保留原始叙事意图，不删除剧情，不改变因果关系
- 如果原镜头承载背景、心理、动机、悬念，拆分后必须明确分配到某个子镜头
- 每个新镜头仍保持 6-12 秒，聚焦单一视觉节拍（visual beat）：一个场景、一个动作阶段、一个对话回合或一个情绪落点
- 子镜头之间自然衔接，避免重复整段原文
- 拆分后整集分镜仍然连续、可拍摄、可生成

## 超载判定

```
text_length = len(narration or "") + len(dialogue or "")
is_overload = text_length > 60
```

阈值可在调用时通过 `threshold` 参数调整；默认 60 字。

## 输出格式

可以输出以下两种形式之一：

1. **只替换超载镜头**：返回被拆分的镜头数组，要求字段完整，并明确覆盖原镜头位置
2. **完整分镜替换**：返回整集完整分镜数组，其中超载镜头已被替换为多个子镜头，其余镜头保持不变

推荐优先使用形式 2，即一次性保存整集完整分镜，确保 `storyboard_number` 连续、不重复。

## 字段要求

每个拆分后的新镜头必须完整填写：

- `title`：3-8 字镜头标题
- `shot_type`：远景/全景/中景/近景/特写
- `angle`：平视/仰视/俯视/侧拍/背面
- `movement`：固定/推镜/拉镜/摇镜/跟镜/移镜
- `location`：地点，与 scenes 中已有地点保持一致
- `time`：时间段，与 scenes 中已有时间保持一致
- `character_ids`：当前镜头涉及的角色 ID 列表，来自角色列表
- `action`：角色具体动作与表演
- `dialogue`：该镜头的实际对白；无对白可置空
- `narration`：该镜头的旁白；无旁白可置空
- `description`：镜头概述，便于人阅读
- `result`：镜头结束时的画面结果或状态变化
- `atmosphere`：氛围、光线、色调、环境感受
- `image_prompt`：静态画面提示词
- `video_prompt`：动态视频提示词，按 3 秒分段
- `bgm_prompt`：配乐风格
- `sound_effect`：关键音效
- `duration`：6-12 秒
- `scene_id`：若能匹配已有场景，必须填写正确 scene_id

## 视频提示词格式

```
0-3秒：<location>咖啡厅</location>，近景，<role>小明</role>低头看手机。
<n>3-6秒：<location>咖啡厅</location>，全景，<role>小红</role>推门走入。
```

## 拆分策略

参考 `make-comics` 的 storyboard 思路：先把连续叙事拆成 visual beats，再为每个 beat 生成独立镜头。按以下优先级寻找拆分点：

1. 角色动作变化（拿起 → 查看 → 放下）
2. 对话自然停顿（一句对白结束、对方回应前）
3. 情绪或节奏转折（惊讶、沉默、决定）
4. 空间位置转移（进门、转身、走近）

如果一个超载镜头同时包含旁白和对白，优先把旁白与对白分离开到不同子镜头，避免声画信息重叠。

如果一条旁白包含两个场景或动作阶段，必须拆开。例如“赵磊开车来接我，停在羊汤店门口”应拆成“车来接人/抵达”和“羊汤店门口或店内重逢”，而不是保留为一个镜头。

## 画面方向

- 横屏 (16:9)：电影化宽幅构图，强调远景、横向运动
- 竖屏 (9:16)：使用“竖屏9:16手机短视频构图”，中近景为主、纵向运动、适合手机观看；不要写横屏、宽幅、宽银幕
- `image_prompt` 和 `video_prompt` 需要体现对应画面比例和构图方式

## 使用步骤

1. 调用 `read_storyboard_context` 读取剧本、角色、场景、已有分镜
2. 遍历每个镜头，计算 `narration + dialogue` 长度
3. 对超载镜头按语义拆分，生成子镜头
4. 为所有新镜头补全字段
5. 调用 `save_storyboards` 保存整集分镜
